Features

Sinobu provides a simple logging facility.

Avoid Boilerplate Code
Logging is performed using static methods like I#info(Object). Therefore, you don't have to create a logger insntace for each class.

void log() {
    I.info("Hello Logging");
}

Garbage Less
Logging operations are highly optimized so that no new objects are created unless call stack information is used.

High Performance
Designed for speed, often outperforming other common logging libraries.

Basic Usage

Basic methods for logging messages at different levels. These methods accept a single message object, which can be Object (using its toString() representation), Supplier which is evaluated lazily only if the log level is enabled, or Throwable (printing stack traces).

Alternatively, most methods accept a String category as the first argument, followed by the message object (e.g., I#info(String, Object)), to direct the log to a specific category configuration. If no category is provided, a default 'system' category is used.

Method Description
I#trace(Object)I#trace(String, Object) Log message at TRACE level.
I#debug(Object)I#debug(String, Object) Log message at DEBUG level.
I#info(Object)I#info(String, Object) Log message at INFO level.
I#warn(Object)I#warn(String, Object) Log message at WARNING level.
I#error(Object)I#error(String, Object) Log message at ERROR level.

Example with category : I.info("database", "Connection established.")

Configuration

Logging behavior can be configured using I#env(String, Object). Configuration keys typically follow the pattern category.property (or just .property for the default logger), where category is the name specified in logging calls (e.g., "database" in the example above).

Output Destinations

Configure where log messages are sent. By default, logs might go to the console depending on the underlying java.lang.System.Logger implementation, but kiss allows explicit configuration. Multiple destinations can be configured for the same logger category.

Property Key Value Type Description
category.console Level Enables console logging for the specified
category at or above the given level.
category.file Level Enables file logging for the specified category
at or above the given level.

File Logging Options

Options specific to file logging when category.file is enabled. Log files are typically named category<Date>.log (e.g., database2024-10-27.log).

Property Key Value Type Description
category.dir Path or String Specifies the directory where log files
for this category should be created.
category.append boolean If true, appends to existing log files for this
category. If false (default), overwrites.
category.rotate int Number of past daily log files to keep for this
category. Older files are deleted. 0 disables rotation. Default is 90.

Formatting Options

Options related to the formatting of log messages for a specific category.

Property Key Value Type Description
category.caller Level Includes caller info for messages in this
category at or above the given level. Can impact performance.

Log Levels

Standard logging levels from java.lang.System.Logger.Level. When a level is set for an output destination (e.g., database.console = Level.INFO), only messages logged to that Category with that level or higher (INFO, WARNING, ERROR) will be written to that destination.

This filtering happens efficiently before message formatting or supplier evaluation, contributing to performance and low GC pressure when levels are disabled.

Available Levels (in increasing order of severity):

  • Level#ALL
  • Level#TRACE
  • Level#DEBUG
  • Level#INFO
  • Level#WARNING
  • Level#ERROR
  • Level#OFF
